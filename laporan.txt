3. Teknologi
Untuk mendukung pengembangan aplikasi RDTR Filter - Sistem Filtering Ketentuan Kegiatan dan Penggunaan Lahan di Kabupaten Bintan, berikut adalah teknologi-teknologi yang digunakan dalam proses pengembangan aplikasi:

________________________________________
1. Front-End (Client-Side)
• Remix Framework
  o Deskripsi: Remix adalah framework React full-stack yang memungkinkan pengembangan aplikasi web dengan server-side rendering (SSR) dan client-side navigation yang optimal. Remix menyediakan fitur loader untuk data fetching dan routing yang efisien.
  o Kegunaan: Digunakan sebagai framework utama untuk membangun aplikasi web dengan performa tinggi, SEO-friendly, dan pengalaman pengguna yang responsif.

• ReactJS
  o Deskripsi: ReactJS adalah pustaka JavaScript untuk membangun antarmuka pengguna (UI) yang dinamis dan interaktif. React memudahkan pengembangan komponen yang dapat digunakan kembali dengan state management yang efisien.
  o Kegunaan: Digunakan untuk membangun komponen UI seperti filter sidebar, activity list, search bar, dan komponen interaktif lainnya dalam aplikasi RDTR Filter.

• Tailwind CSS
  o Deskripsi: Tailwind CSS adalah framework CSS utility-first yang memungkinkan pengembangan antarmuka dengan penulisan kelas CSS minimal dan fleksibel, serta mendukung responsive design.
  o Kegunaan: Membantu mempercepat pengembangan antarmuka dengan kelas-kelas yang sudah tersedia, menciptakan desain yang konsisten dan responsif untuk berbagai ukuran layar.

• TypeScript
  o Deskripsi: TypeScript adalah superset dari JavaScript yang menambahkan static typing, memungkinkan deteksi error lebih awal dan pengembangan yang lebih aman.
  o Kegunaan: Digunakan untuk memastikan type safety dalam pengembangan aplikasi, mendefinisikan interface untuk data struktur, dan meningkatkan maintainability kode.

________________________________________
2. Back-End (Server-Side)
• Node.js
  o Deskripsi: Node.js adalah runtime JavaScript yang memungkinkan eksekusi JavaScript di server-side, menyediakan environment untuk menjalankan aplikasi web.
  o Kegunaan: Digunakan sebagai runtime environment untuk menjalankan aplikasi Remix dan menangani server-side operations.

• Express.js
  o Deskripsi: Express.js adalah framework web minimalis untuk Node.js yang menyediakan fitur routing, middleware, dan HTTP utilities.
  o Kegunaan: Digunakan untuk menangani HTTP requests, routing, dan middleware dalam aplikasi server.

• File System (JSON)
  o Deskripsi: Sistem penyimpanan data menggunakan file JSON untuk menyimpan data RDTR (Rencana Detail Tata Ruang) dan BSB (Bagan Struktur Bangunan).
  o Kegunaan: Menyimpan data aktivitas, zona, dan regulasi dalam format JSON yang mudah dibaca dan diproses oleh aplikasi.

________________________________________
3. Tools dan Utilities

  o Deskripsi: Vite adalah build tool yang menyediakan development server yang cepat dan bundling yang efisien untuk aplikasi modern.
  o Kegunaan: Digunakan untuk development server, hot module replacement, dan building aplikasi untuk production.

  o Kegunaan: Memastikan kode mengikuti best practices dan bebas dari error potensial.

4. Substansi/Materi Perencanaan
Materi Kegiatan Sistem RDTR Filter untuk Koordinasi Pelaksanaan Penataan Ruang Kabupaten Bintan yaitu pengembangan aplikasi web yang memungkinkan filtering dan pencarian ketentuan kegiatan berdasarkan zona dan kode regulasi. Sistem ini mendukung koordinasi pelaksanaan penataan ruang yang melibatkan Dinas Pekerjaan Umum Penataan Ruang dan Pertanahan Kabupaten Bintan serta berbagai bidang dan dinas terkait.

Berikut adalah tugas-tugas yang telah dilaksanakan dalam pembuatan sistem RDTR Filter untuk mendukung kegiatan perencanaan dan pengendalian ruang di Kabupaten Bintan:

1. Pengembangan Aplikasi RDTR Filter V.1.0.0
   Aplikasi RDTR Filter dikembangkan menggunakan Remix framework dengan React dan TypeScript, menyediakan sistem filtering ketentuan kegiatan dan penggunaan lahan berdasarkan zona dan kode regulasi. Aplikasi ini memiliki fitur pencarian zona dengan dropdown searchable, filter regulasi dengan dukungan multi-select, pencarian aktivitas berdasarkan kata kunci, pengurutan data (ascending/descending), dan ekspor data ke format text, CSV, dan Excel. Interface dirancang responsif dengan Tailwind CSS untuk mendukung penggunaan di desktop dan mobile, serta menyediakan dua sumber data (Trikora dan BSB) yang dapat dipilih pengguna.

2. Pengembangan Fitur HTML Parser
   Fitur HTML Parser memungkinkan pengguna untuk mengurai konten HTML dan mengekstrak daftar aktivitas beserta kategorinya secara otomatis. Parser dapat mengenali struktur HTML dengan class 'itbxListTitle' untuk kategori dan 'itbxList-text' untuk aktivitas, mengkonversi teks ke format title case, dan mengelompokkan aktivitas berdasarkan kategori. Fitur ini dilengkapi dengan fungsi pencarian, pengurutan (ascending, descending, original), penyembunyian/penampilan kategori, dan kemampuan copy daftar aktivitas baik secara keseluruhan maupun per kategori.

3. Pengembangan Sistem Filter dan Pencarian Lanjutan
   Sistem filter dikembangkan dengan kemampuan filtering berdasarkan zona menggunakan dropdown searchable dengan keyboard navigation (arrow keys, enter, escape), filter regulasi dengan dukungan single selection dan multi-select menggunakan checkbox, serta pencarian real-time berdasarkan nama aktivitas. Sistem ini juga menyediakan informasi detail regulasi dengan deskripsi lengkap, penanganan URL parameters untuk bookmarking dan sharing, dan reset otomatis filter ketika mengubah sumber data.

4. Pengembangan Fitur Export dan Utilitas
   Fitur export memungkinkan pengguna untuk mengekspor hasil filter ke berbagai format: text (dengan header informasi zona dan regulasi), CSV (format tabular untuk spreadsheet), dan Excel (format .xls dengan tab separator). Setiap export dilengkapi dengan metadata seperti tanggal export, total kegiatan, dan informasi filter yang diterapkan. Aplikasi juga menyediakan utilitas copy to clipboard untuk daftar aktivitas dan dukungan fallback untuk browser yang tidak mendukung clipboard API.

5. Pengembangan Interface Responsif dan User Experience
   Interface aplikasi dirancang dengan pendekatan mobile-first menggunakan Tailwind CSS, menyediakan sidebar filter yang dapat ditampilkan/disembunyikan di mobile dengan overlay, navigation yang mudah digunakan dengan breadcrumb dan link antar halaman, serta feedback visual untuk aksi pengguna (copy success, loading states). Aplikasi juga dilengkapi dengan welcome screen yang informatif ketika belum ada filter yang dipilih dan error handling untuk kasus data tidak ditemukan atau gagal dimuat.
